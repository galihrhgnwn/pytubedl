Overview

PytubeDL API

A self-hosted REST API for fetching YouTube video metadata, stream listings, chapters, key moments, captions, and direct file downloads — built with Python, Flask, and pytubefix.

All endpoints use GET with query parameters. Responses are JSON unless the endpoint streams binary file content.

Authentication

The API uses Google OAuth to bypass YouTube bot detection. Tokens are cached on disk — login only happens once interactively on the VPS.

Run the script below once on the VPS before starting the API (or use setup.sh, which does this automatically). Check OAuth status anytime via GET / (health endpoint).
bash — one-time login 
mkdir -p tokens
python3 -c "
from pytubefix import YouTube
yt = YouTube(
    'https://youtu.be/dQw4w9WgXcQ',
    use_oauth=True,
    allow_oauth_cache=True,
    token_file='./tokens/oauth.json'
)
_ = yt.streams
print('OAuth OK — token saved.')
"

Audio Merge (ffmpeg)

YouTube only serves progressive streams (video + audio in one file) up to 360p — occasionally 720p on older uploads. Anything higher (720p, 1080p, 1440p, 4K) only exists as adaptive/DASH streams, where video and audio are separate files with no sound on the video track alone.

To solve this transparently, every download endpoint that accepts a resolution now does the following automatically:

StepWhat happens
1Check if a progressive stream exists at the requested resolution. If yes, serve it directly — fastest path, already has audio.
2If not, find the matching video-only (adaptive) stream at that exact resolution.
3Find the best available audio-only stream (highest abr) from /api/info.
4Download both temporarily, then mux them with ffmpeg (-c:v copy — stream copy, no re-encoding, so it's fast and lossless for video). Audio is re-encoded to AAC only when the source is Opus/WebM, since that container can't be copied directly into MP4.
5Serve the merged .mp4 file as the response, then delete the temporary files.
This means requesting 720p, 1080p, or higher will always include audio — no more silent video files. Use merge_audio=false on any download endpoint to opt out and get the raw video-only stream instead.
Merging requires ffmpeg installed on the server (apt install ffmpeg). setup.sh installs it automatically. If ffmpeg is missing, high-resolution requests fall back to the best progressive stream available (usually 360p) or return a 503 explaining how to fix it.

Check ffmpeg status

curl
curl "{BASE}/"
json response
{
  "status": "ok",

Errors

Error responses always return JSON with an error key and an optional detail field.

json 
{ "error": "Human-readable message", "detail": "Optional extra detail" }
400Bad Request — missing required parameter or invalid YouTube URL
403Forbidden — video is age-restricted, private, region-blocked, or members-only
404Not Found — video/resolution unavailable, itag not found, or caption code not found
500Server Error — unexpected exception in pytubefix, ffmpeg, or Flask
503Service Unavailable — YouTube bot detection, PO token required, or ffmpeg missing for a merge that's needed

GET /api/streams

Stream List

Returns a filtered list of streams. More granular than the streams field in /api/info — useful for building custom filters.

Query Parameters

ParamTypeDescription
urlrequiredstringYouTube URL
typeoptionalstringall · progressive · adaptive · audio · video — default: all
resolutionoptionalstringe.g. 720p
mime_typeoptionalstringe.g. video/mp4
fpsoptionalintegere.g. 30
abroptionalstringAudio bitrate e.g. 128kbps
video_codecoptionalstringe.g. avc1.640028
audio_codecoptionalstringe.g. mp4a.40.2
is_dashoptionalbooleanInclude/exclude DASH streams
is_drcoptionalbooleanInclude/exclude stable-volume (dynamic range control) streams
order_byoptionalstringAttribute to sort by, e.g. resolution, abr
descoptionalbooleanSort descending — default: true
curl — audio only, sorted by bitrate
curl "{BASE}/api/streams?url=https://youtu.be/dQw4w9WgXcQ&type=audio&order_by=abr"

GET /api/download

Download by Itag

Streams the file directly as a downloadable attachment. Get the itag from /api/info. If the itag points to a video-only (adaptive) stream, audio is merged in automatically via ffmpeg.

Query Parameters

ParamTypeDescription
urlrequiredstringYouTube URL
itagrequiredintegerStream ID from /api/info
merge_audiooptionalbooleanAuto-merge audio if itag is video-only — default: true
Returns binary with Content-Disposition: attachment. Filename derived from video title.
curl — 1080p itag, auto-merged with audio
curl -OJ "{BASE}/api/download?url=https://youtu.be/dQw4w9WgXcQ&itag=137"
curl — raw video-only, no merge
curl -OJ "{BASE}/api/download?url=https://youtu.be/dQw4w9WgXcQ&itag=137&merge_audio=false"
GET /api/download/highest

Download Highest Quality

Downloads the highest resolution available — including 4K/1440p adaptive streams, automatically merged with the best audio via ffmpeg. Pass progressive=true to restrict to progressive-only (no merge, capped around 360–720p).

Query Parameters

ParamTypeDescription
urlrequiredstringYouTube URL
progressiveoptionalbooleanRestrict to progressive streams only — default: false
mime_typeoptionalstringe.g. video/mp4
Default behavior now reaches true maximum quality (e.g. 4K) by merging video-only + audio. This is a change from progressive-only behavior in earlier versions.
curl
curl -OJ "{BASE}/api/download/highest?url=https://youtu.be/dQw4w9WgXcQ"

GET /api/download/resolution

Download by Resolution

Downloads at the exact resolution requested — 144p through 4320p (8K, when the source provides it). Resolutions above what's available as progressive are automatically merged with audio via ffmpeg. See Audio Merge for how this works.

Query Parameters

ParamTypeDescription
urlrequiredstringYouTube URL
resrequiredstring144p · 240p · 360p · 480p · 720p · 1080p · 1440p · 2160p (4K)
merge_audiooptionalbooleanAuto-merge if needed — default: true
If you request a resolution with no matching stream at all (progressive or adaptive), you'll get a 404 with an available_resolutions list so you know what to try instead.
curl — 720p, auto-merged with audio
curl -OJ "{BASE}/api/download/resolution?url=https://youtu.be/dQw4w9WgXcQ&res=720p"
curl — 1080p video-only, no audio merge
curl -OJ "{BASE}/api/download/resolution?url=https://youtu.be/dQw4w9WgXcQ&res=1080p&merge_audio=false"
GET /api/download/audio

Download Audio Only

Downloads the best audio-only stream and serves it with the correct audio MIME type and file extension. YouTube internally labels audio as audio/mp4 or audio/webm, but this endpoint remaps those to proper music formats.

Format Mapping

format=ContainerCodecMIME typeExtension
m4aMP4AAC (mp4a.40.2)audio/mp4.m4a
opusWebMOpusaudio/webm.webm

Query Parameters

ParamTypeDescription
urlrequiredstringYouTube URL
formatoptionalstringm4a (default) · opus
M4A (AAC) is natively playable on all Apple devices and most music players. Opus delivers slightly better quality at the same bitrate but requires a compatible player.
curl — download M4A
curl -OJ "{BASE}/api/download/audio?url=https://youtu.be/dQw4w9WgXcQ&format=m4a"
curl — download Opus
curl -OJ "{BASE}/api/download/audio?url=https://youtu.be/dQw4w9WgXcQ&format=opus"
GET /api/download/dubbed NEW

Download Dubbed Audio Track

Some YouTube videos ship with auto-dubbed or creator-provided alternate-language audio tracks. This downloads one of those tracks by name (check streams.dubbed in /api/info for available names).

Query Parameters

ParamTypeDescription
urlrequiredstringYouTube URL
namerequiredstringTrack name/language, e.g. Spanish (Latin America)
curl
curl -OJ "{BASE}/api/download/dubbed?url=https://youtu.be/dQw4w9WgXcQ&name=Spanish"
Not all videos have dubbed tracks. If none exist, streams.dubbed in /api/info will be an empty array and this endpoint returns 404.
